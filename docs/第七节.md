#### 一、Markdown 基础介绍

Markdown 是一种**轻量级标记语言**，用简单的符号标记文本格式，易读易写，无需复杂操作即可生成结构化文档，广泛用于笔记、博客、README、文档说明等场景。

- 核心特点：纯文本编写，兼容所有文本编辑器，可转换为 HTML、PDF 等格式。
- 常用编辑器：Typora、VS Code（装 Markdown 插件）、Notion、Obsidian 等。
**示例**：

markdown

```
# 一级标题（最多1个，文档主标题）
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

**效果说明**：

- 一级标题字号最大，六级最小；
- 建议层级清晰，避免跳过层级（如直接从一级到四级），提升文档可读性。

##### 2. 段落与换行

**语法格式**：

- 段落：直接输入文本，不同段落之间留**一个空行**即可分隔；
- 强制换行：行尾加 **2 个及以上空格** + 回车，或用 `<br>` 标签。
    
    **示例**：

markdown

```
这是第一个段落，Markdown的段落不需要额外标记，直接写文字即可。
（空行）
这是第二个段落，与上一段落之间有一个空行分隔。

这一行文字末尾加两个空格  
这一行会在上一行的下方换行显示（行尾两个空格+回车）。

这一行用<br>标签换行<br>这是换行后的内容。
```

**效果说明**：无空行的文本会被识别为同一个段落，强制换行仅改变行位置，不产生段落间距。

##### 3. 列表（有序 / 无序 / 嵌套）

### （1）无序列表

**语法格式**：用 `*`/`+`/`-` 开头，符号与文字之间加空格，三种符号效果一致。

**示例**：

markdown

```
- 无序列表项1
* 无序列表项2
+ 无序列表项3
```

### （2）有序列表

**语法格式**：用「数字 + `.`」开头，数字顺序不影响显示（最终按 1、2、3 排序），符号与文字之间加空格。

**示例**：

markdown

```
1. 有序列表项1
2. 有序列表项2
3. 有序列表项3
```

### （3）列表嵌套

**语法格式**：嵌套项前加 **4 个空格** 或 **1 个 Tab**，支持多级嵌套。

**示例**：

markdown

```
- 一级无序列表
    1. 二级有序列表
    2. 二级有序列表
        - 三级无序列表
- 一级无序列表2
```

**效果说明**：嵌套列表需严格缩进，否则会被识别为同级列表。

##### 4. 强调（粗体 / 斜体 / 删除线）

**语法格式**：

- 斜体：文字包裹在 `*` 或 `_` 中（单对）；
- 粗体：文字包裹在 `**` 或 `__` 中（双对）；
- 粗斜体：文字包裹在 `***` 或 `___` 中（三对）；
- 删除线：文字包裹在 `~~` 中。
    
    **示例**：

markdown

```
*斜体文本*
_斜体文本_
**粗体文本**
__粗体文本__
***粗斜体文本***
~~删除线文本~~
```

**效果**：

_斜体文本_、**粗体文本**、_**粗斜体文本**_、~~删除线文本~~。

##### 5. 链接（超链接）

### （1）行内式链接（最常用）

**语法格式**：`[链接文字](链接地址 "可选的悬浮标题")`，悬浮标题鼠标悬浮时显示。

**示例**：

markdown

```
[百度](https://www.baidu.com "跳转到百度首页")
```

**效果**：[百度](https://www.baidu.com "跳转到百度首页")。

### （2）参考式链接（适合多重复用）

**语法格式**：先定义链接标识，再在文本中引用，适合文档中多次使用同一链接的场景。

**示例**：

markdown

```
这是[百度][1]的链接，再次引用[百度][1]。

[1]: https://www.baidu.com "百度首页"
```

**效果**：与行内式一致，仅写法不同，便于统一管理链接地址。

### （3）锚点链接（跳转到文档内指定位置）

**语法格式**：`[跳转文字](#标题内容)`，标题内容需小写，空格替换为 `-`，特殊字符去掉。

**示例**：

markdown

```
[跳转到「表格」章节](#6-表格)
```

**效果**：点击链接会跳转到文档内「6. 表格」的位置（需标题存在）。

##### 6. 图片

### （1）基础图片语法

**语法格式**：`![图片加载失败时的替代文字](图片地址 "可选的悬浮标题")`（比链接多一个 `!`）。

**示例**：

markdown

```
![测试图片](https://picsum.photos/200/150 "这是一张测试图片")
```

**效果**：

![](data:image/svg+xml,%3csvg%20xmlns=%27http://www.w3.org/2000/svg%27%20version=%271.1%27%20width=%27400%27%20height=%27256%27/%3e)![image](https://picsum.photos/200/150 "这是一张测试图片")

### （2）图片加链接（点击图片跳转）

**语法格式**：将图片语法嵌套在链接语法中。

**示例**：

markdown

```

```

**效果**：点击图片会跳转到百度首页。

##### 7. 代码块（行内 / 块级）

### （1）行内代码

**语法格式**：用反引号 `` ` `` 包裹代码片段，适合单行代码 / 变量。

**示例**：

markdown

```
这是行内代码：`print("Hello World")`，用于标记小段代码。
```

**效果**：这是行内代码：`print("Hello World")`，用于标记小段代码。

### （2）块级代码（代码块）

**语法格式**：用 `包裹多行代码，开头` 后可指定语言（如 python、html），实现语法高亮。

**示例**：

##### 8. 引用（块引用）

**语法格式**：用 `>` 开头，支持多级嵌套（多个 `>`），可嵌套其他语法（列表、链接等）。

**示例**：

markdown

```
> 一级引用文本
>> 二级引用文本（嵌套）
>>> 三级引用文本
> 
> 引用中嵌套列表：
> - 列表项1
> - 列表项2
```

**效果**：

> 一级引用文本
> 
> > 二级引用文本（嵌套）
> > 
> > > 三级引用文本
> 
> 引用中嵌套列表：
> 
> - 列表项 1
> - 列表项 2

##### 9. 表格

**语法格式**：

- 第一行：表头，列之间用 `|` 分隔；
- 第二行：分隔线，用 `-` 表示，`:` 控制对齐（`:---` 左对齐、`---:` 右对齐、`:---:` 居中）；
- 第三行及以后：表格内容，列之间用 `|` 分隔。
    
    **示例**：

markdown

```
| 姓名 | 年龄 | 城市 |
| :--- | :---: | ---: |
| 张三 | 25 | 北京 |
| 李四 | 30 | 上海 |
| 王五 | 28 | 广州 |
```

**效果**：

|姓名|年龄|城市|
|:--|:-:|--:|
|张三|25|北京|
|李四|30|上海|
|王五|28|广州|

##### 10. 分割线

**语法格式**：一行中用 3 个及以上 `-`/`*`/`_`，符号之间可空格，无其他字符。

**示例**：

markdown

```
---
***
___
```

**效果**：

---

---

---

##### 11. 进阶语法（常用）

### （1）任务列表

**语法格式**：`- [ ]` 表示未完成，`- [x]` 表示已完成（注意空格）。

**示例**：

markdown

```
- [ ] 学习Markdown标题语法
- [x] 学习Markdown列表语法
- [ ] 学习Markdown表格语法
```

**效果**：

- [ ]  学习 Markdown 标题语法
- [x]  学习 Markdown 列表语法
- [ ]  学习 Markdown 表格语法

### （2）注脚

**语法格式**：

- 注脚标记：`文本[^注脚标识]`；
- 注脚内容：`[^注脚标识]: 注脚的具体内容`。
    
    **示例**：

markdown

```
Markdown是轻量级标记语言[^1]，易读易写。

[^1]: 由John Gruber于2004年创建。
```

**效果**：

Markdown 是轻量级标记语言 [^1]，易读易写。

[^1]: 由 John Gruber 于 2004 年创建。

#### 三、语法使用注意事项

1. 符号与文字之间**必须加空格**的语法：标题（`#`）、列表（`-`/`1.`）、引用（`>`）；
2. 反引号 `` ` `` 是英文符号，避免用中文全角符号导致语法失效；
3. 表格的 `|` 前后可加空格，不影响显示，建议对齐提升可读性；
4. 不同编辑器对 Markdown 的支持略有差异（如注脚、任务列表），以实际编辑器效果为准。

### 总结

1. Markdown 核心语法以「简单符号 + 空格」为基础，重点掌握标题、列表、链接 / 图片、代码块、表格这 5 类高频用法；
2. 语法关键规则：层级靠缩进、分隔靠空行、符号与文字间需加空格；
3. 进阶语法（任务列表、注脚）可根据场景选用，优先保证文档结构清晰、易读。
